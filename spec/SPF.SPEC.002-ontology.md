# Спецификация онтологии Pack'а

> Как Pack должен описывать свою онтологию: обязательные секции, формат, правила.

---

## 1. Назначение

Каждый Pack содержит файл **онтологии** — единый реестр всех понятий и типов сущностей предметной области.

Онтология отвечает на три вопроса:
1. **Какие типы сущностей** существуют в этом домене? (виды + коды)
2. **Что означают ключевые понятия** домена? (глоссарий)
3. **Как типы и понятия связаны?** (отношения)

### Чем онтология НЕ является

| Онтология — это НЕ | Где это живёт |
|---------------------|--------------|
| Bounded context (границы, scope) | `01A-bounded-context.md` |
| Различения (A ≠ B) | `01B-distinctions.md` |
| Правила кодирования | `SPF.SPEC.001` |
| Карта связей между сущностями | `07-map/` |

Онтология дополняет bounded context (что входит/не входит) **определениями** и дополняет различения (A ≠ B) **полным глоссарием**.

---

## 2. Размещение

```
<pack>/
├── 01-domain-contract/
│   ├── 01A-bounded-context.md   # Границы и scope
│   ├── 01B-distinctions.md      # Различения (A ≠ B)
│   └── 01C-ontology.md          # ← Онтология
```

Файл: `01C-ontology.md` в папке `01-domain-contract/`.

## 3. Обязательные секции

### 3.1. Реестр видов сущностей

Полный список типов, используемых в Pack: базовые (из SPF) + расширенные (определённые этим Pack).

**Формат таблицы:**

| Код | Вид | EN | Определение | ≠ (что это НЕ) | Источник |
|-----|-----|----|-------------|-----------------|----------|
| `M` | Метод | Method | ... | ≠ сценарий, ≠ инструмент | SPF (базовый) |
| `ARCH` | Архитектура | Architecture | ... | ≠ реализация | Pack (расширенный) |

**Обязательные столбцы:**
- **Код** — код вида (из SPF.SPEC.001)
- **Вид** — название на русском
- **EN** — название на английском
- **Определение** — 1-2 предложения
- **≠** — от чего отличать (синонимы-ловушки)
- **Источник** — `SPF (базовый)` или `Pack (расширенный)`

### 3.2. Глоссарий домена

Ключевые понятия предметной области с определениями.

**Формат:**

| Термин | Определение | Связан с |
|--------|-------------|----------|
| Цифровой двойник | Вычислительная модель, синхронизированная с объектом | DP.CONCEPT.001 |

**Требования:**
- Только понятия, существенные для понимания домена
- Каждое понятие — 1-2 предложения
- Ссылка на сущность Pack, если есть

### 3.3. Отношения между видами

Как виды сущностей связаны друг с другом.

**Формат:**

| Субъект | Отношение | Объект | Пример |
|---------|-----------|--------|--------|
| Метод | порождает → | Рабочий продукт | DP.M.001 → DP.WP.001 |
| Failure Mode | нарушает ← | Метод | DP.FM.001 ← DP.M.001 |

**Стандартные типы отношений:**

| Отношение | Описание |
|-----------|----------|
| порождает → | Метод порождает рабочий продукт |
| нарушает ← | Failure mode — нарушение метода или различения |
| реализует → | ИИ-система реализует метод |
| описывает → | Карта описывает связи между сущностями |
| расширяет → | Расширенный вид уточняет базовый |

Pack может определять дополнительные типы отношений, специфичные для домена.

## 4. Опциональные секции

### 4.1. Онтологические уровни

Если Pack использует иерархию типов (общий → частный), описать её:

```
ИИ-система (AGENT)
├── ИИ-агент (orientation: system)
└── ИИ-ассистент (orientation: human)
```

### 4.2. Кросс-Pack связи

Понятия, заимствованные или совместно используемые с другими Pack'ами.

| Понятие этого Pack | Связанный Pack | Понятие там |
|--------------------|---------------|-------------|
| Индикатор | PD (Personal Dev) | PD.CHR.* |

---

## 5. Правила

1. **Полнота.** Онтология включает ВСЕ виды сущностей, используемые в Pack (и базовые, и расширенные)
2. **Согласованность.** Определения в онтологии = определения в bounded context. Расхождений быть не должно
3. **Не дублирование.** Различения (A ≠ B) остаются в `01B-distinctions.md`. Онтология ссылается на них, но не повторяет
4. **Кодирование.** Коды видов — по SPF.SPEC.001
5. **Актуальность.** При добавлении новой сущности в Pack — обновить онтологию

## 6. Связь с другими спецификациями

| Спецификация | Связь |
|-------------|-------|
| SPF.SPEC.001 (кодирование) | Онтология использует коды из SPEC.001 |
| 01A-bounded-context | Онтология определяет понятия; bounded context определяет границы |
| 01B-distinctions | Онтология ссылается на различения; различения — в своём файле |
| 00-pack-manifest | Манифест регистрирует расширенные виды; онтология даёт полные определения |
| 07-map | Карта показывает связи между конкретными сущностями; онтология — между видами |
